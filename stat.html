The Idea

Two problems, one engine

Large models are expensive in two very different ways. This project tackles both by refactoring proven technology (llama.cpp CPU/GPU offload) and adding one proprietary, measured module (SSCC).

🖥️ Problem 1 — the model doesn't fit

A 30B model in Q8 is ~31 GiB. It won't sit in a single consumer GPU. The Hybrid-Offload Engine splits the layers between GPU and CPU RAM so the model runs anyway — and it runs 85% faster than pure CPU.

🧬 Problem 2 — the context is bloated

RAG pipelines feed the model huge, mostly-irrelevant context. Every token costs VRAM locally and money on paid APIs. SSCC keeps only the salient chunks — −40 to −69% tokens with the answer intact.

Result 1 · Inference Speed

Hybrid offload beats pure CPU by 85%

We ran the same 30B model in three configurations on identical hardware. Naively using all logical cores (SMT) is actually slower than physical cores alone — a classic compute-bound penalty. Offloading 25 layers to the GPU is the clear winner.

Result 2 · Context Compression

SSCC — keep the needle, drop the hay

Instead of blindly truncating context, SSCC scores every chunk by a salience formula that blends semantic relevance, information density (amplitude) and recency (velocity), then keeps only the top ones.

score = cos(query, chunk) · (1 + α·Amp) · (γ + (1−γ)·Vel)

α = 0.35 (density weight) · γ = 0.70 (recency floor) · cos via a dependency-free numpy TF-IDF embedder

Result 4 · Retention Stress-Test

The needle survives — 500 trials, no model

This is the model-free proof. Across 500 randomized trials we hide one critical fact among 30 same-domain distractors and measure how often it survives compression. SSCC and pure cosine keep it every single time; truncation is a coin-flip at best.

% of trials where the critical fact survives compression

Why this matters for the accuracy number Because retention is 100%, the ceiling on SSCC accuracy is set entirely by the model's ability to read its own context — not by information loss. That's a fixable, model-side problem, not a flaw in the compression.
